Customization

Customization of the application branding, including name, images, and colors, is centralized in a single file: frontend/public/branding/theme.yaml.

1. Where to configure branding?​

The main file for branding configuration is:

frontend/public/branding/theme.yaml

In this file, you can define:

  • The application name (appName)
  • Main colors (colors)
  • Branding images (images)

Configuration Example:​

appName: "OctoCast"

colors:
  primary: "#1dac78"      # Primary color (buttons, links, accents)
  secondary: "#344054"    # Secondary color (texts, borders)
  background: "#f9fafb"   # Background color
  accent: "#2ecc71"       # Accent color (alerts, highlights)
  error: "#dc2626"        # Error color (error messages)

images:
  logo: "/branding/ochoIconFull.svg"
  default_miniature_image: "/branding/exemple/image_tuile_event.png"
  add: "/branding/add.svg"
  plus: "/branding/plus.svg"
  search: "/branding/search.svg"
  cross: "/branding/cross.svg"
  user_placeholder: "/branding/persona.png"

2. Typing and validation​

The schema of this YAML file is defined in:

frontend/src/branding/type.ts

The BrandingConfig interface ensures that file properties are properly typed and validated in TypeScript code.

3. Automatic color generation​

The branding system simplifies color management by automatically generating variants. From 4-5 base colors, it produces over 50 variants for the entire application.

How it works:​

  • Light colors (50-400): Progressive blend with white
  • Base color (500): Exact color defined in the file
  • Dark colors (600-900): Progressive blend with black

These variants are generated by the generateColorVariants function in:

frontend/src/utils/colorUtils.ts

Generated CSS Variables:​

For each base color, the system creates CSS variables:

  • --theme-color, --theme-color-50 to --theme-color-900
  • --bg-color, --bg-color-50 to --bg-color-900
  • --accent-color, --accent-color-50 to --accent-color-900
  • --error-color, --error-color-50 to --error-color-900

4. Usage in components​

useBranding Hook​

The useBranding hook is defined in:

frontend/src/hooks/useBranding.ts

It allows to:

  • Load the theme.yaml file
  • Generate color variants via generateColorVariants
  • Dynamically inject CSS variables into the DOM
  • Return the configuration for use in React components

ColorPreview Component​

The ColorPreview component is used to display generated color variants. It is defined in:

frontend/src/components/ReworkComponents/generic/ColorPreview/ColorPreview.tsx

5. Administration Panel​

The administration panel allows you to modify branding in real-time. It is accessible at the URL: /admin (administrator rights required).

Features:​

  • Modify the application name
  • Change main colors
  • Preview gradients in real-time
  • Modify branding images
  • Save configuration on the server

The corresponding file is:

frontend/src/pages/adminPanel/adminPanel.tsx
Detailed Guide

For a complete and detailed guide to the administration panel with explanations of each field and feature, see the Administration Panel Guide.

6. Summary​

  • 4-5 defined colors → 50+ CSS variables automatically generated
  • Automatic gradient system (variants 50-900)
  • Administration panel for real-time modification
  • All branding centralized in theme.yaml
  • Modular and typed architecture with TypeScript
  • Automatic reload after save to apply changes
